Por que dados sintéticos?

Dados reais de crédito são confidenciais e sujeitos à LGPD. Neste curso, geramos uma carteira fictícia com 10.000 contratos usando regras estocásticas que embutem relações propositalmente realistas:

# src/data_generation.py — função principal def generate_portfolio(n_loans=10_000, seed=42): rng = np.random.default_rng(seed) # Score baixo → maior logit_pd → maior PD logit_pd = ( -4.5 - 0.006 * (score_current - 500) + 0.8 * (dti > 0.40) + 0.40 * num_delinq + 0.02 * dpd ) pd_true = sigmoid(logit_pd) default_flag = (rng.random(n) < pd_true)

Relações embutidas

  • Score ↑ → PD ↓ (melhor pagador)
  • DTI ↑ → PD ↑ (comprometimento de renda)
  • Inadimplências passadas ↑ → PD ↑
  • Garantia imobiliária → LGD ↓ (mais recuperação)
  • LTV alto → LGD ↑ (garantia insuficiente)
  • Alta utilização → CCF ↑ (clientes sacam mais)

Segmentação de Carteira e Abordagem Coletiva

O IFRS 9 permite calcular a ECL de forma coletiva (pooled) — agrupando contratos com características de risco similares — em vez de individualmente (quando não houver informação suficiente a nível de contrato).

Critérios de Segmentação

  • Tipo de produto: imobiliário, cartão, veículo — risco completamente distinto
  • Vintage (safra): contratos com mesma data de originação têm comportamento homogêneo
  • Rating / Score: faixas de PD agrupadas
  • Região geográfica: concentração por estado, impacto de choques locais
  • Canal de originação: digital, agência, correspondente bancário
  • Porte do tomador (PF vs PJ): PME vs Large Corporate

Granularidade recomendada

IASB ED/2013/3: Um segmento deve conter pelo menos 30–50 defaults observados para que os parâmetros PD/LGD sejam estatisticamente robustos. Segmentos muito pequenos resultam em alta variância paramétrica.
# Segmentação típica no modelo df["segment"] = ( df["loan_type"] + "_" + pd.cut(df["pd_12m"], bins=[0,.02,.10,.30,1], labels=["low","med","high","vhigh"]) .astype(str) )

Análise de Safra (Vintage Analysis)

A análise de safra rastreia o comportamento de inadimplência de grupos de contratos originados no mesmo período. É a base para calibrar PDs e entender a qualidade de originação ao longo do tempo.

Curva de Safra: DR(meses_desde_originação) DR₁₂ = % contratos em default até o 12º mês DR₂₄ = % contratos em default até o 24º mês Maturação: ponto em que a curva estabiliza

Uma safra com DR₁₂ maior que as anteriores sinaliza deterioração dos critérios de crédito na originação.

Application Score × Behavioral Score

Application Score

Usado na originação: prevê o risco do tomador antes de conceder o crédito. Baseia-se em dados cadastrais e bureau de crédito (Serasa, SCR, etc.).

  • Score bureau (Serasa Experian, SPC)
  • Histórico de consultas de crédito
  • Renda declarada e comprovada
  • Tempo de relacionamento com o banco
  • DTI (dívida / renda)

Behavioral Score

Usado no monitoramento contínuo: reflete o comportamento atual do cliente com base em dados transacionais internos.

  • Padrão de pagamento (dias de atraso)
  • Evolução da utilização do limite
  • Saques no cheque especial frequentemente
  • Volume de transações no débito/crédito
  • Relação saldo/limite ao longo do tempo
IFRS 9 usa behavioral score para monitorar SICR — o score application já é histórico; o behavioral é dinâmico.

Principais Aprendizados

  • A carteira sintética reproduz relações realistas entre features e risk drivers
  • O staging usa regras de ASRC baseadas em score, DPD e DTI combinados
  • A maior parcela (~60%) fica no Stage 1 — adimplente sem deterioração
  • Crédito Empresarial tem a maior taxa de default; Veículos a menor

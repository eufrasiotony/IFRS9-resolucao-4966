Para quem está começando — Entendendo o modelo de EAD

O EAD responde: quando o cliente der default, qual será o saldo devedor que o banco vai enfrentar? Para um empréstimo simples, é fácil. Para um cartão de crédito, pode ser bem diferente do saldo de hoje.

Empréstimo pessoal (não-rotativo)
Limite: R$ 10.000
Saldo restante hoje: R$ 7.000
O saldo só diminui com o tempo — nunca sobe.
EAD = R$ 7.000 (sem modelo necessário)
Cartão de crédito (rotativo)
Saldo atual: R$ 500 | Limite disponível: R$ 4.500
Antes de dar default, o cliente pode usar mais do limite.
CCF = 70% → EAD = R$500 + 70% × R$4.500
EAD = R$ 3.650 (modelo necessário)
Atenção: o CCF NÃO é uma probabilidade, e o EAD final é um valor em R$!
O CCF ∈ [0, 1] é um fator de conversão de crédito — representa o quanto do limite disponível o cliente tende a usar antes do default. CCF = 0 → não saca nada a mais; CCF = 1 → usa 100% do limite disponível.
O EAD é o resultado final em reais (R$), não um percentual de chance.

O que é a Exposure at Default?

O EAD é o valor que o banco espera ter em risco no momento em que o contrato entrar em default. Para produtos não-rotativos (ex: empréstimo pessoal com prazo fixo), o saldo devedor é determinístico. Para produtos rotativos (ex: cartão de crédito, cheque especial), o cliente pode sacar até o limite — o que exige um modelo.

$$\text{EAD} = \text{Saldo Atual} + CCF \times \text{Limite Não Utilizado}$$ onde: CCF = Credit Conversion Factor ∈ [0, 1] Para produtos não-rotativos: CCF = 0 (EAD = saldo atual)

Credit Conversion Factor (CCF)

O CCF captura o comportamento de uso de limite antes do default. Clientes em dificuldade financeira tendem a puxar o limite disponível antes de dar default ("drawdown behavior").

Metodologia

O CCF NÃO é uma probabilidade — o EAD é um valor monetário

O CCF ∈ [0, 1] é um fator de conversão de crédito (credit conversion factor), não uma chance de evento.
CCF = 0 → o cliente não usa mais nada do limite antes do default
CCF = 1 → o cliente usa 100% do limite disponível antes do default

Assim como o logit do LGD, a transformação sigmoid é usada apenas para manter o CCF em [0, 1].
O resultado final do modelo é o EAD em R$: EAD = Saldo + CCF × Limite Disponível.

O CCF também é uma variável bounded [0,1]. Aplicamos Ridge na transformação logit, similar ao LGD. O GBM modela diretamente o CCF com clipping.

Desafio: R² Baixo

EAD real tem R² baixo — e isso é esperado!

O CCF é altamente estocástico. Mesmo em bancos com décadas de dados, R² de 0.05–0.15 para CCF bruto é comum. O que importa é que o CCF médio por segmento esteja bem calibrado para o ECL agregado.
Prática regulatória: Muitos bancos usam CCFs médios históricos por produto/bucket ao invés de modelos individuais por contrato.

LEQ × CCF: diferença conceitual

Os termos são frequentemente confundidos, mas têm definições distintas na literatura de crédito:

CCF — Credit Conversion Factor

$$CCF = \frac{EAD - Saldo_{atual}}{Limite_{nao\_utilizado}}$$ Mede: qual fração do limite disponível será sacada até o default?

CCF = 1 → cliente usa 100% do limite disponível antes de dar default. CCF = 0 → não usa nada além do que já utilizou.

LEQ — Loan Equivalent Exposure

$$LEQ = \frac{EAD - Saldo_{atual}}{\text{Compromisso total} - Saldo_{atual}}$$ Mede em relação ao total comprometido, não apenas ao não-utilizado.

LEQ é preferido quando a exposição total comprometida (on + off balance) é o denominador relevante. Basileia II usava LEQ; Basileia III tendeu a CCF.

Na prática IFRS 9: Usa-se o CCF ou abordagem de comprometimento total. O padrão não prescreve uma fórmula única — exige que o EAD represente a melhor estimativa da exposição no momento do default.

Drawdown Behavior: Comportamento Pré-Default

Uma das descobertas mais consistentes na literatura de risco de crédito é que clientes em deterioração financeira sacam os limites disponíveis antes de dar default ("drawdown effect"). Isso é especialmente intenso em cartões e cheque especial.

$$\text{Utilização}_t = \frac{\text{Saldo}_t}{\text{Limite}_t}$$ Observação empírica: $Utilização_{t-12} \approx 40\%$ $Utilização_{t-6} \approx 65\%$ $Utilização_{t-3} \approx 81\%$ $Utilização_{t=0} \approx 94\%$ (no default)

Implicações para modelagem:

  • Usar utilização atual como feature no modelo de CCF
  • Segmentar por faixa de utilização: baixa (<30%), média, alta (>70%)
  • CCF médio de clientes com utilização alta já próxima de 1.0 é menor (pouco limite disponível para sacar adicionalmente)
  • Comportamento varia por tipo de produto e perfil de cliente
Paradoxo do CCF: clientes com utilização mais baixa tendem a ter CCF mais alto — há mais limite para sacar.

EAD por Contrato × EAD por Segmento

Modelo Individual (contrato a contrato)

Estima o CCF para cada contrato usando features individuais. Alta variância, difícil de calibrar — mas permite granularidade total para o ECL contrato a contrato.

R² individual geralmente < 0.10. O modelo "acerta a direção" mas o erro pontual é alto. Aceitável se o ECL agregado estiver bem calibrado.

CCF Médio por Segmento (pooled)

Calcula o CCF médio histórico por célula: produto × faixa de utilização × vintage. Mais estável, regulatoriamente aceito, amplamente usado em bancos de médio porte.

Stress de EAD e Floor Regulatório

Em cenários de stress, o comportamento de drawdown se intensifica. Bancos centrais (BCB, ECB) exigem stress de EAD nos testes de estresse.

$$EAD_{stress} = Saldo + CCF_{stress} \times Limite_{disponivel}$$ $$CCF_{stress} = \min(1,\ CCF_{base} \times (1 + \delta_{stress}))$$ δ_stress típico: +25% a +50% sobre o CCF base

Floors e Limites Mínimos

  • Floor do EAD: EAD ≥ saldo atual (nunca abaixo do que já está em risco)
  • Floor de CCF: Basileia III: CCF ≥ 0 para compromissos revogáveis; CCF ≥ 10% mesmo para linhas sem saque histórico
  • EAD mínimo em IFRS 9: inclui principal + juros accrued + fees capitalizados
  • Cashflows futuros: para crédito revolvente, o IFRS 9 permite EAD projetado ao longo da vida esperada do produto (behavioral maturity)
Behavioral maturity (IFRS 9): Para cartão de crédito sem vencimento definido, o banco deve estimar a vida esperada com base no comportamento histórico de renovação/cancelamento. Tipicamente 2–5 anos.

Principais Aprendizados

  • EAD é simples para produtos não-rotativos (saldo = EAD); o desafio é nos rotativos
  • O CCF captura o drawdown de limite antes do default — difícil de prever individualmente
  • R² baixo para EAD/CCF é esperado e não invalida o modelo — calibração agregada importa mais
  • Para fins de ECL, muitos bancos usam CCF médios históricos por segmento ao invés de modelos ML
  • A IFRS 9 aceita abordagens simplificadas para EAD quando a exposição é relativamente estável
